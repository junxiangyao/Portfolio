Overview

As an exploration of visualizing information in a virtual reality environment, this project attempts to combine visual complexity with a convenient and manageable information presentation system. It focuses on displaying the comparison and composition patterns in the dataset.

This project aims to design a universal pattern to visualize time-oriented data, including numeric and text-based datasets. This project visualized the daily check-out data of Star Wars movies from the Seattle Public Library (SPL) and the news headlines from the Seattle Times on the corresponding dates from 2005 to 2018.

Features

1.Mapping Calendar

The dates from January 1st, 2005, to December 31st, 2018, are mapped on the floor. Thus, there will be a corresponding date to any location on the floor. The user can retrieve the daily data by working/teleporting to the location of a particular date.



2.Checking Daily Data

Each data node floating above a specific date contains the daily data of a particular category (differentiated by different colors). The larger the daily quantitative data is, the higher the node will be. The user can hover over the node to check on the detail.



3.Minimap

Because the size of this structure is too large for the user to observe the overview of the visualization, a minimap with a radius of 15 centimeters was attached to the left controller. This minimap contains a smaller replica of the full-scaled visualization, which enables the user to observe the overview with greater ease. They can also click on the minimap to teleport in the space.



4.Interface for Scaling

To fill the significant gap in size between the minimap and the full-scale structure, a scaling interface that would be used to shrink the scene was added to recognize more details in the overview. Users can also use this interface to teleport and check the daily data.

Data Visualization

Data Base

To explore image-making in virtual reality and figure out a solution to visualize multidimensional data, I used both numeric and text-based datasets.

The data used in this project are daily check-out data of the Star Wars movies retrieved from the Seattle Public Library (SPL) and the local news headlines on corresponding dates from the Seattle Times. I retrieved the desired numeric dataset in MySQL Workbench and saved them in CSV files.

Previous Version

This project is based on the Star Wars Nebula, an earlier data visualization work I created in 2016, using the same dataset from SPL. The goal was to reveal the pattern of the check-out times and the borrowing duration of the items related to the movies in the Star Wars franchise and to design an innovative way to present numeric data.

In the visualization, two levels of temporal granularity (year and month) were used in designing the form based on the format of the data. These data were divided into six categories containing the check-out data of one movie in the franchise, respectively.

New Form

For the new version, I used a smaller granularity of data, day, combined with text-based data in the visualization. So I designed a new form.

In the coordinate system, I used concentric circles to divide the floor. Each ring contains data from one year. And those circles were also divided into 12 sections clockwise to map the data from 12 months. Each of these sections will be further divided into smaller parts evenly for daily data.



The data was divided into different categories and represented by different colors. The data nodes from each category will be mapped on the floor. The node's elevation from the base plane represents the corresponding daily check-out data of that category. The data nodes are connected to their neighbors from the same categories by lines.

Advantages of the New Form

Observable Overview

To show the overview of the visualization, a compact form to visualize the data is ideal since the user should be able to see the whole picture at a glance. A circular shape saves more space than other formats.

Enhancing Patterns

Periodic patterns are more apparent in a spiral or circular structure. This structure compares the data from the same date in different years, which is a goal inherited from the earlier version. The lines connecting all neighbors of the nodes bring more comparisons and thus intensify the pattern visually.

3D Interaction Design

Major Interaction Tasks

According to Ben Shneiderman's famous paper, The Eyes Have It: A Task by Data Type Taxonomy for Information Visualizations, there are 7 major interaction tasks in data visualization applications in general. This project used the includes the first 4 tasks: Overview, Zoom, Filter, and Details-on-Demand. And I realized these interaction tasks in VR based on the form and the space.



1. Overview

The minimap attached to the Oculus left controller will provide an overview of the entire visualization. The user could observe the overview just like looking at a wristwatch to check the time. Once the user found any interesting pattern, he/she can teleport to that location to check the detail by clicking on the minimap.



2. Zoom

To check the detail, the user can walk or teleport in the visualization. I also added a scaling interface. The user could use it to shrink the large-scale graphics down to anywhere from ten to thirty percent of its original size.



3. Filter

The users can use the buttons to show and hide lines based on the month, year, and movie they belong to. These buttons and UI components were located on a virtual plane attached to the left hand. This virtual plane will remain hidden unless the user activates it using the controller.



4. Details-on-Demand

The user could use the ray-pointer to point to the data nodes to check the detail of the data, similar to the mouse-hovering and tooltips on desktop applications.



Process

Starting with Developing a Processing Prototype

I started with converting my sketch and ideas into code. I used Processing to build the visualization prototype. From the process, I learned the limitations of interacting with multidimensional data visualization through a 2D screen, as zooming and changing focus are very tricky with mouse and keyboard.

This could easily be solved by using VR and 3D Interaction.

Rebuilding in Unity 3D with the FPS Control

Then I transferred the Processing code into C# in Unity 3D. I used the first-person shooting (FPS) camera from Unity's developing asset and the keyboard control to prototype the experience of walking around inside the visualization.

The result showed that changing focus became easier since users now have the freedom to move around and look at where ever they want in the space.

Developing Teleporting Features in VR

As the interaction logic changed drastically from 2D screen to 3D space, I experimented and developed several C# scripts for teleporting, including the minimap. And I connected Oculus Rift to Unity to check the results.

Teleporting is essential to this project's interaction as it is the main method for users to retrieve data. It can also reduce the possibility of cyber-sickness.

Developing Features in VR

I merged the data visualization scene and the teleporting scene into the final VR space. Next, I wrote the C# scripts for the remaining interaction tasks with Oculus Rift connected.

Testing

I did two rounds of iteration with two rounds of testing. The second round of testing is conducted during an exhibition, I asked the visitors to do some tasks in the VR space and fill out a google form.

From the 16 forms I received, users didn't feel dizziness bothering them while they were using the application. However, since VR is still a new field that many users may not have yet experienced, there is a learning curve for people to use VR applications. Feedback suggests adding more tutorials, like instruction animations, at the beginning.


Reflections

Future Steps
This project was done before I studied interaction design at California College of the Arts. Thus, there are several usability issues revealed from testing that could be solved using human-centered design methods. Also, I couldn't find better datasets that contain clearer insights into human behavior due to time constraints. I will explore more datasets and topics and update this project to add more reflective values to this project.

Learning by Doing
The process of developing this project is my journey of learning Unity 3D and its C# scripts. This is the first time I started to work on an entirely new software and coding language without much preparation. I enjoyed this learning by doing process as I learned things fast this way. I think it is similar to the logic behind rapid prototyping: constant and early feedback from trying is essential to make progress.
